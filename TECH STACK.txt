1. Frontend (The App)

    Framework: React + TypeScript

        Why: Fast performance, type safety prevents bugs, huge ecosystem.

    Styling: TailwindCSS

        Why: Tiny CSS file size (good for 2G networks), easy to build consistent UI.

    Animations: Framer Motion (Light usage)

        Why: Smooth transitions for "Success Ticks" and page slides without lagging low-end phones.

    Architecture: Progressive Web App (PWA)

        Why: Installable on Android without the Play Store, works offline, updates instantly.

2. Backend (The Cloud)

    Core Platform: Supabase

        Why: It provides Database, Auth, and API in one suite. Perfect for offline sync.

    Database: PostgreSQL

        Why: Reliable relational data for complex financial ledgers.

    Logic: Supabase Edge Functions

        *Why:*Server-side logic (like complex rate calculations or heavy reports) runs here to save phone battery.

    File Storage: Supabase Storage

        Why: Storing generated PDF bills and user profile images.

3. Offline Engine (Crucial)

    Local Database: IndexedDB

        Recommendation: Use a wrapper like Dexie.js or RxDB to make managing IndexedDB easier.

    Sync Logic: Local Mutation Queue

        How it works: When the user saves milk offline, it goes into a "queue". When internet returns, the queue sends data to Supabase.

    Conflict Resolution: Latest Valid Write

        Why: Simple rule to decide which data wins if edited on two devices.

4. Security & Access

    Authentication: Supabase Auth (Mobile OTP)

    Data Protection: Row Level Security (RLS)

        Rule: A dairy owner can only see their own rows. A farmer can only see their own ledger.

5. Recommended Libraries (For Implementation)

(These are standard standard industry choices to fit your PRD requirements)

    PDF Generation: @react-pdf/renderer (For generating bills on the phone).

    Forms: react-hook-form (For fast data entry without re-renders).

    Validation: zod (To ensure FAT/SNF numbers are valid before saving).

    Date Handling: date-fns (Lightweight date formatting for ledgers).
